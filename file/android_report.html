三种 APK 构建方案
A

GitHub Actions 自动构建 ⭐ 推荐

难度：简单

将代码推送到 GitHub，Actions 自动在云端 Ubuntu 机器上构建 APK，构建完成后作为 Artifact 下载。

# 1. 创建 GitHub 仓库并推送 git init git add . git commit -m "Android app" git remote add origin https://github.com/你的账号/vlpr.git git push -u origin main # 2. 在 GitHub Actions 页面等待构建 # 3. 下载 Artifacts 中的 APK
  • ✅ 无需本地安装任何 Android 工具
  • ✅ 免费（公开仓库），每月 2000 分钟免费
  • ⚠️ 首次构建约 30-60 分钟
B

WSL2 本地构建

难度：中等

在 Windows 的 WSL2 Ubuntu 环境中安装 Buildozer，本地编译生成 APK。

# WSL2 Ubuntu 中执行 sudo apt update && sudo apt install -y \ git zip openjdk-17-jdk python3-pip cmake pip3 install buildozer==1.5.0 cython==0.29.36 # 进入项目目录 cd /mnt/c/Users/HUAWEI/Desktop/\ FinalDesign/Python_VLPR-master buildozer android debug
  • ✅ 本地构建，不依赖网络 CI
  • ✅ 可随时重新构建
  • ⚠️ 首次需下载 SDK/NDK (~4GB)
  • ⚠️ 需要 WSL2 已安装 Ubuntu
C

云端 Linux 服务器

难度：较高

在阿里云/腾讯云等 Ubuntu 实例上构建，适合需要稳定构建环境的场景。

# 在 Linux 服务器上 sudo apt update sudo apt install -y openjdk-17-jdk \ python3-pip git zip cmake pip3 install buildozer cython git clone https://github.com/你/vlpr cd vlpr buildozer android debug # 下载生成的 APK scp user@server:~/vlpr/bin/*.apk .
  • ✅ 环境稳定，方便复用
  • ⚠️ 需要有自己的服务器
  • ⚠️ 需要额外费用
GitHub Actions 详细操作步骤
1

注册/登录 GitHub 账号

访问 github.com 注册账号（免费）

2

创建新仓库

点击右上角 "+" → "New repository"，仓库名如 vlpr-android，选择 Public，点击 "Create repository"

3

本地初始化 Git 并推送

在 PowerShell 中执行以下命令（先安装 Git）：

cd "C:\Users\HUAWEI\Desktop\FinalDesign\Python_VLPR-master" git init git add main.py main_android.py img_function.py img_math.py git add img_recognition.py config.py debug.py git add svm.dat svmchinese.dat buildozer.spec git add .github/ git add img/ git commit -m "feat: Android app with Kivy UI" git branch -M main git remote add origin https://github.com/你的账号/vlpr-android.git git push -u origin main
4

查看构建进度

打开 GitHub 仓库页面 → 点击 "Actions" 标签 → 找到刚触发的 workflow → 点击查看实时日志

5

下载 APK

构建成功后（绿色 ✅），页面底部 "Artifacts" 区域点击 vlpr-debug-apk 下载 ZIP，解压得到 APK 文件

打开手机设置 → 隐私/安全 → 开启"允许安装未知来源应用" → 用文件管理器找到 APK → 点击安装 → 授予相机和存储权限

常见问题与解决方案
⚠️
首次构建时间较长
Buildozer 需要下载 Android SDK (~600MB)、NDK (~1GB) 和 Python-for-Android，总计约 30-60 分钟。第二次构建有缓存，只需 5-10 分钟。

🔴 ImportError: No module named 'opencv'

在 buildozer.spec 中，OpenCV 的包名是 opencv（不是 opencv-python），已在新版 spec 中修正。

🔴 中文字符显示为方块

Kivy 默认字体不含中文。如需中文显示，在 spec 中打包中文字体文件（.ttf），并在 Label 中指定 font_name='NotoSansCJK.ttf'。

🔴 APK 安装后闪退

通常是模型文件未正确打包。检查 source.include_patterns 是否包含 svm.dat,svmchinese.dat，已在新版 spec 中明确配置。

🔴 摄像头无法打开

需要在安装后首次启动时授予相机权限，或在手机设置 → 应用 → 车牌识别 → 权限中手动开启。

🔴 构建内存不足 (OOM)

增加 Linux swap 空间：sudo fallocate -l 4G /swapfile && sudo mkswap /swapfile && sudo swapon /swapfile

🟡 APK 体积过大

已通过 source.exclude_dirs = train,test,file,workplace 排除训练数据（约 500MB）。最终 APK 应在 50-80MB 左右。
